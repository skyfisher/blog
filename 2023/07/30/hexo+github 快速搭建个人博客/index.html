10 分钟快速搭建个人博客——基于hexo和github

导语

博客是技术人最好的名片，虽然在 23 年的当下，内容平台日渐丰富，只要有好的内容，基本不缺发布的平台；但它在笔者心中依然有不可替代的作用，笔者认为写博客主要有以下好处：

  • 系统化整理个人的知识，在当前大环境分工日益精细化的情况下，零散的知识点在帮助我们解决复杂的问题上很难发挥作用，体系化的知识梳理后构建的知识体系，则能成为我们解决问题的有力武器；
  • 呈现个人的思考，类比职场中的邮件同步，将文字写下来，本身就是一个深度思考和反复推敲的过程，可以帮助我们深度聚焦在一个具体的问题和场景上，避免泛泛而谈；
  • 沉淀个人的技术影响力，酒香也怕巷子深，闷头造车越做越窄，拥抱开源拥抱分享，则收获的是更宽广的成长之路。

本文是此博客的第一篇，本文的主要内容主要是有两个方面，一是如何基于 hexo+github 快速搭建个人专属博客，二是笔者自己在搭建博客过程碰到的一些问题的解答。

安装 Hexo

Hexo 是一个快速、简洁高效的博客网站，安装过程也非常简单，安装好必须的 node 和 git 后，通过 npm 安装的方式，就可以直接安装 hexo 了。

本地调试环境

本地创建文件目录并初始化，这里我选择了创建和博客站点域名一样的目录文件名，在本机上启动 hexo 之后，默认会运行在 4000 端口可以通过 127.0.0.1:4000/进行访问。

初始的页面如下图所示，第一个 hello world 页面已经准备就绪，接下里只需要配置好 github，将本地的博客的文件推送到远端，我们的站点就相当于已经建好。

配置 Github 托管

最终我们的博客还是需要发布到线上的，这里我们选择托管在 github 上，首先我们需要配置好 ssh 的自动登录

  1. 在 github 页面上打开设置；

  1. 选择 SSH and GPG keys 并点击 new SSH key；

  1. 通过 ssh-keygen在本机生成sshkey，并将生成的 key 的内容内容拷贝到前步的内容中保存；

  1. 仍需要安装 git的部署插件，并修改 hexo 工程文件中的_config.yml内容
  1. 配置完成之后，我们就可以把本地的内容推送到 github 进行托管了

现在就能在站点上看到刚刚写下的这篇文章了。

总结&FAQ

从上述步骤可以看出，快速搭建一个博客站点是非常方便的，不需要我们自己准备服务器、域名等基础设施，就可以快速搭建一个个人站点。剩余的内容就是发挥自己的才智，开启我们的博客之旅了。

Q：如何修改博客的主题?

A: 文件夹中有一个 theme 的目录，下载相关的 hexo 主题到本目录(例如我自己使用的是 next)，然后在_config.yml文件中配置所选的主题即可，相关的主题下载可以参考站点：https://hexo.io/themes/

Q：本地的截图等文件如何上传？

A：这里以使用 typora 来编辑本地文件为例子

  1. 在 source 目录下创建 images 文件夹，/images/filename.png 相关的文件在 hexo 服务器上是默认可以访问到的；

  2. 设置 typora 相关的截图文件存储到固定路径；

  3. 配置 typora 的根目录为 source/，从而在 mkdown 中插入的图片，在本地和在 hexo 的服务器上，能够用同样的路径明加载到图片

    设置的路径为 格式->图像->图片根目录->选择对应的目录
