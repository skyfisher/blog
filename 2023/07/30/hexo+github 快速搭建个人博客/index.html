hexo+github 快速搭建个人博客

导语

本文是此博客的第一篇，读者能够从中获得如下两方面的经验

  • 读者个人对写作的经验经验之谈；
  • 如何基于 hexo+github 快速搭建一个个人专属的播客的经验

个人博客是技术人最好的名片，虽然在 23 年的当下，内容平替日渐丰富，只要有好的内容，基本不缺发布的平台；但个人博客依然是笔者心中的一片净土，撰写个人博客的好处在笔者看来主要体现在：

  • 系统化整理个人的知识：零散的知识在当前分工日益明细的情况下，比较难发挥作用，知识的体系化才能形成凝聚力；
  • 体系化呈现个人的思考：类似工作中的邮件同步，把文字写下来，形成一篇篇幅不算短的文章，本身就是一个深度思考的过程，可以帮助我们很快聚焦在一个具体的问题和场景上；
  • 体系化展现个人的技术影响力：酒香也怕巷子深，闷头造车越做越窄，拥抱开源拥抱分享，则收获的是更宽广的成长之路。

FAQ

Q：如何修改博客的主题?

A: 文件夹中有一个 theme 的目录，下载相关的 hexo 主题到本目录(例如我自己使用的是 next)，然后在_config.yml文件中配置所选的主题即可

Q：本地的截图等文件如何上传

A：这里以使用 typora 来编辑本地文件为例子

  1. 在 source 目录下创建 images 文件夹，/images/filename.png 相关的文件在 hexo 服务器上是默认可以访问到的；

  2. 设置 typora 相关的截图文件存储到固定路径

  3. 配置 typora 的根目录为 source/，从而在 mkdown 中插入的图片，在本地和在 hexo 的服务器上，能够用同样的路径明加载到图片

    设置的路径为 格式->图像->图片根目录->选择对应的目录
